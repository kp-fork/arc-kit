# ArcKit Plugin Hooks

Hook handlers live in this directory and are registered in `hooks.json`. Supported hook events include `SessionStart`, `UserPromptSubmit`, `PreToolUse`, `PostToolUse`, `Stop`, `StopFailure`, `PermissionRequest`, plus the newer `PostCompact` (v2.1.76), `FileChanged`/`CwdChanged`/`TaskCreated` (v2.1.83‚Äì84), `PermissionDenied` (v2.1.89), and `PreCompact` blocking (v2.1.105).

## `if:` Field for Narrow Triggering

Individual hook entries support an `if:` field (Claude Code v2.1.85+) using **permission rule syntax** to narrow triggering and avoid unnecessary Node process spawns. Examples:

- `"if": "Write(/projects/**)"` ‚Äî only fires for Write calls whose resolved path is under `<project-root>/projects/`
- `"if": "Edit(/projects/**)"` ‚Äî same for Edit
- `"if": "Bash(git *)"` ‚Äî only fires for `git` subcommands

Path patterns follow gitignore semantics (`/path` is relative to project root, `//path` is filesystem-absolute). When adding a new hook, prefer narrow `matcher` + `if:` over broad `.*` unless the hook needs to react to every invocation (e.g., `secret-detection` on UserPromptSubmit).

## Provenance Stamping (`provenance-stamp.mjs`)

PostToolUse hook on `Write|Edit` against `projects/**` that injects a `## Build Provenance` block at the end of every ArcKit artefact (`ARC-NNN-*-vN.N.md`). Block is delimited by `<!-- arckit-provenance:start -->` / `<!-- arckit-provenance:end -->` markers and rewritten in place on subsequent runs (idempotent).

The block **complements** the human-authored footer that the generating command writes (`Generated by:`, `AI Model:`, `Generation Context:`, etc.) ‚Äî it carries only fields the model can't authoritatively self-report. It does not duplicate Skill / Model / Generated-on / Project / ArcKit Version / Inputs (those live in the existing footer).

Stamped fields:

- **Build context** (Recipe / Wave / Target / Topic) ‚Äî only present when run via `/arckit:build`, sourced from `projects/{P}/.arckit/state.json`
- **Requested Effort** ‚Äî `effort:` field from the invoking command's YAML frontmatter
- **Effective Effort** ‚Äî computed by parsing the `AI Model:` line from the existing footer and applying the silent-downgrade matrix in `provenance-stamp.mjs` (`MODEL_MAX_EFFORT`). When the model doesn't support the requested level, the value reads e.g. `high (downgraded from max ‚Äî model does not support that level)` ‚Äî the auditable signal that issue #407 was filed for.
- **Stamped at** ‚Äî ISO 8601 timestamp (per write)

If neither effort nor build context is available (e.g. a non-ArcKit command edited the file), the hook skips stamping entirely ‚Äî no empty block. Existing artefacts pre-dating the hook are stamped on the next Write/Edit.

When a stamp is actually written, the hook emits `hookSpecificOutput.updatedToolOutput` (Claude Code v2.1.121+) so the model sees a one-line summary ‚Äî effort requested vs. effective, downgrade reason if any, and (on Write) a confirmation that `docs/manifest.json` was auto-updated by `update-manifest.mjs` upstream. On no-op runs (no harness fields, file unchanged) the hook stays silent and the original tool output is preserved.

## Manifest Auto-Update (`update-manifest.mjs`)

Companion PostToolUse hook on `Write` against `projects/**` that incrementally updates `docs/manifest.json` so the documentation site stays current without a full `/arckit:pages` re-run. On a successful update it emits `hookSpecificOutput.updatedToolOutput` with the document ID and target slot (e.g. `ARC-001-REQ-v1.0 ‚Üí 001-test/documents`). Because `provenance-stamp.mjs` runs after this hook and overwrites its `updatedToolOutput`, the manifest signal is re-surfaced from `provenance-stamp.mjs` whenever a `docs/manifest.json` exists.

## Session Telemetry (`telemetry.mjs` + `session-learner.mjs`)

Lightweight telemetry recorder registered for three events:

- **PostToolUse** (matcher `.*`) ‚Äî records `{ tool, duration_ms }` for every tool call (Claude Code v2.1.119+ `duration_ms` field). Pure-latency records exclude `TaskCreate` and `mcp__govreposcrape__*` calls so the duration histogram isn't polluted by long-running async tools.
- **PostToolUse** (same registration, branched by tool name) ‚Äî records `{ server, tool, args }` for `mcp__govreposcrape__*` calls. Args are sanitised (long strings replaced with length markers, nested objects flattened to `<object>`) so the JSONL stays small.
- **TaskCreated** (matcher `.*`, Claude Code v2.1.84+) ‚Äî records `{ agent }` for every agent spawn via the Task tool.

Events are appended to `.arckit/memory/.telemetry.jsonl` during the session. `session-learner.mjs` reads, summarises, and truncates the file at Stop / StopFailure, adding a single line to each session entry, e.g.:

```text
- **Telemetry:** 47 tool calls (p50=12ms, p95=4200ms) | 3 agents (arckit-research√ó2, arckit-datascout) | MCP: govreposcrape√ó8
```

Telemetry is best-effort ‚Äî any failure (write error, malformed line, missing field) is swallowed silently so it never breaks a session.

When `docs/` exists (i.e. the project has run `/arckit:pages`), `session-learner.mjs` also writes a structured rollup to `docs/telemetry.json` (newer-first, capped at 50 sessions) so the dashboard can render a "Session Telemetry" + "Recent Sessions" panel. Each record contains `{ ts, type, isFailure, commits, filesChanged, artifacts, telemetry: { toolCalls, p50, p95, agents, mcp } }`. The dashboard fetches `telemetry.json` with the same graceful-fallback pattern as `health.json` ‚Äî projects without a `docs/` directory render the dashboard exactly as before.

> **Note on `type: "mcp_tool"` (v2.1.118)**: This Claude Code feature lets a hook *invoke* an MCP tool as its action (alternative to `type: "command"` / `type: "prompt"`). It does **not** filter hooks to fire only on MCP tool calls. ArcKit logs `govreposcrape` calls via the existing tool-name matcher pattern (`mcp__govreposcrape__.*`); no `type: "mcp_tool"` registration is needed for telemetry.
